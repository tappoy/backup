backup <command>
  commands:
    - help               show this message
    - info               show the service status
    - start <vault-dir>  start the service. required password.
    * stop               stop the service
    * get <obj> <dst>    download the object.
    * delete <obj>       remove the object.
    * list               list objects in container.

  (*) need service to be started.

  Path:
    - vault-dir: The path of the vault directory.
    - log file:  /var/log/backup/{info,notice,error}.log
    - pid file:  /var/run/backup.pid
    - pipe file: /var/run/backup.pipe

  This service starts backing up at 2:00 daily.

  Vault:
    Configs of clouds are stored in the vault.
    The vault manage these keys:
      - BACKUP_n
    Like:
      - BACKUP_1
      - BACKUP_2 ...
    Backup to all of clouds these BACKUP_n keys.

    See:
      - https://github.com/tappoy/archive
      - https://github.com/tappoy/vault-cli
